23.What is  SQl ?

1Ô∏è‚É£ SQL Databases (Relational DB)

SQL = Structured Query Language

Data stored in tables (rows & columns)

Schema is fixed (you must define structure before inserting data)

Use SQL queries to interact

Features

Relational (tables linked by relationships)

Strong consistency

ACID properties (Atomicity, Consistency, Isolation, Durability)

Examples

MySQL, PostgreSQL, Oracle, SQL Server


23.1 What is NOSQl ?

2Ô∏è‚É£ NoSQL Databases (Non-relational)

Data stored in flexible formats (JSON documents, key-value, graphs, column stores)

Schema-less (can store different fields in each record)

Often horizontal scalable for large data

Features

Flexible schema

Can handle large, unstructured data

Eventual consistency (depends on DB)

Fast for specific queries

Examples

MongoDB (document-based)

Redis (key-value)

Cassandra (column-based)

24.Types of nosql database  ?

1Ô∏è‚É£ Document-Based Database

Stores data as documents (usually JSON or BSON)

Each document can have different fields

Great for apps with flexible or hierarchical data

Examples: MongoDB, CouchDB


2Ô∏è‚É£ Key-Value Database

Stores data as key-value pairs

Extremely fast for lookup by key

Good for caching and session storage

Examples: Redis, DynamoDB, Riak

"user:123": { "name": "Rayhan", "email": "rayhan@gmail.com" }

3Ô∏è‚É£ Column-Based (Wide-Column) Database

Stores data in columns instead of rows

Efficient for analytical queries on large datasets

Often used in Big Data applications

Examples: Cassandra, HBase

Row Key: 1
Columns: name=Rayhan, email=rayhan@gmail.com, role=user

4Ô∏è‚É£ Graph Database

Stores data as nodes and edges

Great for relationships (social networks, recommendations)

Queries follow connections between data

Examples: Neo4j, ArangoDB, Amazon Neptune

(Rayhan)-[:FRIEND]->(Sara)
(Rayhan)-[:BOUGHT]->(Property1)
